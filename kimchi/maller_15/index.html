Maller's optimization for Kimchi

This document proposes a protocol change for kimchi.

What is Maller's optimization?​

See the section on Maller's optimization for background.

Overview​

We want the verifier to form commitment to the following polynomial:

L(x)=f(x)−ZH(ζ)⋅t(x)L(x) = f(x) - Z_H(\zeta) \cdot t(x)L(x)=f(x)−ZH​(ζ)⋅t(x)

They could do this like so:

com(L)=com(f)−ZH(ζ)⋅com(t)\mathsf{com}(L) = \mathsf{com}(f) - Z_H(\zeta) \cdot \mathsf{com}(t)com(L)=com(f)−ZH​(ζ)⋅com(t)

Since they already know fff, they can produce com(f)\mathsf{com}(f)com(f), the only thing they need is com(t)\mathsf{com}(t)com(t). So the protocol looks like that:

In the rest of this document, we review the details and considerations needed to implement this change in kimchi.

How to deal with a chunked ttt?​

There's one challenge that prevents us from directly using this approach: com(t)\mathsf{com}(t)com(t) is typically sent and received in several commitments (called chunks or segments throughout the codebase). As ttt is the largest polynomial, usually exceeding the size of the SRS (which is by default set to be the size of the domain).

The verifier side​

Thus, the verifier will have to produce split commitments of LLL and combine them with powers of ζn\zeta^nζn to verify an evaluation proof. Let's define LLL as:

L=L0+xnL1+x2nL1+⋯L = L_0 + x^n L_1 + x^{2n} L_1 + \cdotsL=L0​+xnL1​+x2nL1​+⋯

where every LiL_iLi​ is of degree n−1n-1n−1 at most. Then we have that

com(L)=com(L0)+com(xn⋅L1)+com(x2n⋅L2)+⋯\mathsf{com}(L) = \mathsf{com}(L_0) + \mathsf{com}(x^n \cdot L_1) + \mathsf{com}(x^{2n} \cdot L_2) + \cdotscom(L)=com(L0​)+com(xn⋅L1​)+com(x2n⋅L2​)+⋯

Which the verifier can't produce because of the powers of xnx^nxn, but we can linearize it as we already know which xxx we're going to evaluate that polynomial with:

com(L~)=com(L0)+ζn⋅com(L1)+ζ2n⋅com(L2)+⋯\mathsf{com}(\tilde L) = \mathsf{com}(L_0) + \zeta^n \cdot \mathsf{com}(L_1) + \zeta^{2n} \cdot \mathsf{com}(L_2) + \cdotscom(L~)=com(L0​)+ζn⋅com(L1​)+ζ2n⋅com(L2​)+⋯

The prover side​

This means that the prover will produce evaluation proofs on the following linearized polynomial:

L~(x)=1⋅L0(x)+ζn⋅L1(x)+ζ2n⋅L2(x)+⋯\tilde L(x) = 1 \cdot L_0(x) + \zeta^n \cdot L_1(x) + \zeta^{2n} \cdot L_2(x) + \cdotsL~(x)=1⋅L0​(x)+ζn⋅L1​(x)+ζ2n⋅L2​(x)+⋯

which is the same as L(x)L(x)L(x) only if evaluated at ζ\zetaζ. As the previous section pointed out, we will need L~(ζω)\tilde L(\zeta \omega)L~(ζω) and L~(ζω)≠L(ζω)\tilde L(\zeta \omega) \neq L(\zeta \omega)L~(ζω)=L(ζω).

Evaluation proof and blinding factors​

To create an evaluation proof, the prover also needs to produce the blinding factor rLr_{L}rL​ associated with the chunked commitment of:

L~=f~−(ζn−1)t~\tilde L = \tilde f - (\zeta^n - 1) \tilde tL~=f~​−(ζn−1)t~

To compute it, there are two rules to follow:

  • when two commitment are added together, their associated blinding factors get added as well: com(a)+com(b)  ⟹  ra+rb\mathsf{com}(a) + \mathsf{com}(b) \implies r_a + r_bcom(a)+com(b)⟹ra​+rb​
  • when scaling a commitment, its blinding factor gets scaled too: n⋅com(a)  ⟹  n⋅ran \cdot \mathsf{com}(a) \implies n \cdot r_an⋅com(a)⟹n⋅ra​

As such, if we know rfr_frf​ and rtr_trt​, we can compute:

rL~=rf~+(ζn−1)⋅rt~r_{\tilde L} = r_{\tilde f} + (\zeta^n-1) \cdot r_{\tilde t}rL~​=rf~​​+(ζn−1)⋅rt~​

The prover knows the blinding factor of the commitment to ttt, as they committed to it. But what about fff? They never commit to it really, and the verifier recreates it from scratch using:

  1. The commitments we sent them. In the linearization process, the verifier actually gets rid of most prover commitments, except for the commitment to the last sigma commitment Sσ6S_{\sigma6}Sσ6​. (TODO: link to the relevant part in the spec) As this commitment is public, it is not blinded.
  2. The public commitments. Think commitment to selector polynomials or the public input polynomial. These commitments are not blinded, and thus do not impact the calculation of the blinding factor.
  3. The evaluations we sent them. Instead of using commitments to the wires when recreating fff, the verifier uses the (verified) evaluations of these in ζ\zetaζ. If we scale our commitment com(z)\mathsf{com}(z)com(z) with any of these scalars, we will have to do the same with rzr_zrz​.

Thus, the blinding factor of L~\tilde LL~ is (ζn−1)⋅rt~(\zeta^n-1) \cdot r_{\tilde t}(ζn−1)⋅rt~​.

The actual protocol changes​

Now here's how we need to modify the current protocol:

  1. The evaluations f(ζ),f(ζω),t(ζ),t(ζω)f(\zeta), f(\zeta \omega), t(\zeta), t(\zeta \omega)f(ζ),f(ζω),t(ζ),t(ζω) (and their corresponding evaluation proofs) don't have to be part of the protocol anymore.
  2. The prover must still send the chunked commitments to ttt.
  3. The prover must create a linearized polynomial L~\tilde LL~ by creating a linearized polynomial f~\tilde ff~​ and a linearized polynomial t~\tilde tt~ and computing: L~=f~+(ζn−1)⋅t~\tilde L = \tilde f + (\zeta^n-1) \cdot \tilde tL~=f~​+(ζn−1)⋅t~
  4. While the verifier can compute the evaluation of L~(ζ)\tilde L(\zeta)L~(ζ) by themselves, they don't know the evaluation of L~(ζω)\tilde L(\zeta \omega)L~(ζω), so the prover needs to send that.
  5. The verifier must recreate com(L~)\mathsf{com}(\tilde L)com(L~), the commitment to L~\tilde LL~, themselves so that they can verify the evaluation proofs of both L~(ζ)\tilde L(\zeta)L~(ζ) and L~(ζω)\tilde L(\zeta\omega)L~(ζω).
  6. The evaluation of L~(ζω)\tilde L(\zeta \omega)L~(ζω) must be absorbed in both sponges (Fq and Fr).

The proposal is implemented in #150 with the following details:

  • the L~\tilde LL~ polynomial is called ft.
  • the evaluation of L~(ζ)\tilde L(\zeta)L~(ζ) is called ft_eval0.
  • the evaluation L~(ζω)\tilde L(\zeta\omega)L~(ζω) is called ft_eval1.
