Understanding the implementation of the f(ζ)=ZH(ζ)t(ζ)f(\zeta) = Z_H(\zeta) t(\zeta)f(ζ)=ZH​(ζ)t(ζ) check

Unlike the latest version of vanilla PlonK\plonkPlonK that implements the final check using a polynomial opening (via Maller's optimization), we implement it manually. (That is to say, Izaak implemented Maller's optimization for 5-wires.)

But the check is not exactly f(ζ)=ZH(ζ)t(ζ)f(\zeta) = Z_H(\zeta) t(\zeta)f(ζ)=ZH​(ζ)t(ζ). This note describes how and why the implementation deviates a little.

What's f and what's missing in the final equation?​

If you look at how the evaluation of f(z)f(z)f(z) is computed on the prover side (or the commitment of fff is computed on the verifier side), you can see that f is missing two things:

  • the public input part
  • some terms in the permutation
