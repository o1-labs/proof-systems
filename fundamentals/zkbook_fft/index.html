Fast Fourier Transform (FFT)

Fast Fourier Transform (FFT)​

This section describes how the Cooley-Tukey fast Fourier transform works. As we learned in the previous section, the key is to select evaluation points that yield an efficient FFT algorithm.

Specifically, say we have ω∈F\omega \in Fω∈F such that ωn=1\omega^{n} = 1ωn=1, and ωr≠1\omega^r \neq 1ωr=1 for any 0<r<n0 < r < n0<r<n.

Put another way, all the values 1,ω,ω2,ω3,…,ωn−11, \omega, \omega^2, \omega^3, \dots, \omega^{n-1}1,ω,ω2,ω3,…,ωn−1 are distinct and ωn=1\omega^n = 1ωn=1.

Put yet another way, the group generated by ω\omegaω inside F×F^\timesF× (written ⟨ω⟩\langle \omega \rangle⟨ω⟩) has size nnn.

We call such an ω\omegaω a primitive nnn-th root of unity.

Suppose we have an ω\omegaω which is a primitive 2k2^k2kth root of unity and let Ak={1,ω,…,ω2k−1}A_k = \{ 1, \omega, \dots, \omega^{2^k - 1} \}Ak​={1,ω,…,ω2k−1}.

The FFT algorithm will let us compute interpAk\mathsf{interp}_{A_k}interpAk​​ for this set.

Actually, it is easier to see how it will let us compute the evalAk\mathsf{eval}_{A_k}evalAk​​ algorithm efficiently.

We will describe an algorithm FFT(k,ω,f)\mathsf{FFT}(k, \omega, f)FFT(k,ω,f) that takes as input

  • k∈Nk \in \Nk∈N
  • ω∈F\omega \in Fω∈F a primitive 2k2^k2kth root of unity
  • f∈F[x]<2kf \in F[x]_{< 2^k}f∈F[x]<2k​ in dense coefficients form (i.e., as a vector of coefficients of length nnn).

and outputs the vector of evaluations

[f(1),f(ω),f(ω2)…,f(ω2k−1)][f(1), f(\omega), f(\omega^2) \dots, f(\omega^{2^k - 1})][f(1),f(ω),f(ω2)…,f(ω2k−1)]

and does it in time O(k2k)O(k 2^k)O(k2k) (which is to say, nlog⁡nn \log nnlogn if n=2kn = 2^kn=2k).

Notice that naively, computing each evaluation f(ωi)f(\omega^i)f(ωi) using the coefficients of fff would require time O(n)O(n)O(n), and so computing all nnn of them would require time O(n2)O(n^2)O(n2).

The algorithm FFT(k,ω,f)\mathsf{FFT}(k, \omega, f)FFT(k,ω,f) can be defined recursively as follows.

If k=0k = 0k=0, then ω\omegaω is a primitive 111st root of unity, and fff is a polynomial of degree 000. That means ω=1\omega = 1ω=1 and also fff is a constant c∈Fc \in Fc∈F. So, we can immediately output the array of evaluations [c]=[f(1)][c] = [f(1)][c]=[f(1)].

If k>0k > 0k>0, then we will split fff into two polynomials, recursively call FFT\mathsf{FFT}FFT on them, and reconstruct the result from the recursive calls.

To that end, define f0f_0f0​ to be the polynomial whose coefficients are all the even-index coefficients of fff and f1f_1f1​ the polynomial whose coefficients are all the odd-index coefficients of fff. In terms of the array representation, this just means splitting out every other entry into two arrays. So that can be done in time O(n)O(n)O(n).

Now let's analyze the time complexity. Let T(n)T(n)T(n) be the complexity on an instance of size nnn (that is, for n=2kn = 2^kn=2k).

Looking back at what we have done, we have done

  • O(n)O(n)O(n) for computing f0f_0f0​ and f1f_1f1​
  • two recursive calls, each of size n/2n / 2n/2
  • O(n)O(n)O(n) for computing the powers of ω\omegaω
  • O(n)O(n)O(n) for combining the results of the recursive calls

In total, this is O(n)+2T(n/2)O(n) + 2 T(n / 2)O(n)+2T(n/2). Solving this recurrence yields T(n)=O(n)⋅logn=O(nlog⁡n)T(n) = O(n) \cdot log n = O(n \log n)T(n)=O(n)⋅logn=O(nlogn). Basically, there are log⁡n\log nlogn recursions before we hit the base case, and each step takes time O(n)O(n)O(n). □\square□

Now, in practice there are ways to describe this algorithm non-recursively that have better concrete performance, but that's out of scope for this document. Read the code if you are interested.

Using the FFT algorithm to compute interpAk\mathsf{interp}_{A_k}interpAk​​​

So far we have a fast way to compute evalAk(f)\mathsf{eval}_{A_k}(f)evalAk​​(f) all at once, where AkA_kAk​ is the set of powers of a 2k2^k2kth root of unity ω\omegaω. For convenience let n=2kn = 2^kn=2k.

Now we want to go the other way and compute a polynomial given an array of evaluations. Specifically, nnn evaluations [f(x0),f(x1),...,f(xn−1)]\left[f(x_0), f(x_1), ..., f(x_{n - 1})\right][f(x0​),f(x1​),...,f(xn−1​)] uniquely define a degree n−1n - 1n−1 polynomial. Takeaways​

  • Polynomials can be represented as a list of coefficients or a list of evaluations on a set AAA

  • If the set AAA is the set of powers of a root of unity, there are time O(nlog⁡n)O(n \log n)O(nlogn) algorithms for converting back and forth between those two representations

  • In evaluations form, polynomials can be added and multiplied in time O(n)O(n)O(n)
