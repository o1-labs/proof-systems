Half Gate

The Half-Gate optimization further reduces the number of ciphertexts from 333 to 222. This is also the protocol used in zkOracles, and more details of this algorithm will be given in this subsection. The algorithm is presented in the ZRE15 paper.

We first describe the basic idea behind half-gate garbling. Let's say we want to compute the gate a∧b=ca\wedge b = ca∧b=c. With the Free-XOR optimization, let (Xa,Xa⊕Δ)(X_a,X_a\oplus \Delta)(Xa​,Xa​⊕Δ) and (Xb,Xb⊕Δ)(X_b,X_b\oplus\Delta)(Xb​,Xb​⊕Δ) denote the input wire labels to this gate, and (Xc,Xc⊕Δ)(X_c,X_c\oplus \Delta)(Xc​,Xc​⊕Δ) denote the output wire labels, with Xa,Xb,XcX_a,X_b,X_cXa​,Xb​,Xc​ each encoding 000.

Half-gate garbling contains two case: when the garbler knows one of the inputs, and when the evaluator knows one of the inputs.

Garbler half-gate. Considering the case of an AND\andAND gate a∧b=ca\wedge b = ca∧b=c, where a,ba,ba,b are intermediate wires in the circuit and the garbler somehow knows in advance what the value aaa will be.

When a=0a = 0a=0, the garbler will garble a unary gate that always outputs 000. The label table is as follows.

bbbccc
XbX_bXb​XcX_cXc​
Xb⊕ΔX_b\oplus \DeltaXb​⊕ΔXcX_cXc​

Then the garbler generates two ciphertexts: H(Xb)⊕Xc\sH(X_b)\oplus X_cH(Xb​)⊕Xc​ H(Xb⊕Δ)⊕Xc\sH(X_b\oplus\Delta)\oplus X_cH(Xb​⊕Δ)⊕Xc​

When a=1a = 1a=1, the garbler will garble a unary identity gate. The label table is as follows.

bbbccc
XbX_bXb​XcX_cXc​
Xb⊕ΔX_b\oplus \DeltaXb​⊕ΔXc⊕ΔX_c\oplus \DeltaXc​⊕Δ

Then the garbler generates two ciphertexts:

H(Xb)⊕Xc\sH(X_b)\oplus X_cH(Xb​)⊕Xc​ H(Xb⊕Δ)⊕Xc⊕Δ\sH(X_b\oplus\Delta)\oplus X_c\oplus \DeltaH(Xb​⊕Δ)⊕Xc​⊕Δ

Since aaa is known to the garbler, the two cases shown above could be unified as follows:

H(Xb)⊕Xc\sH(X_b)\oplus X_cH(Xb​)⊕Xc​ H(Xb⊕Δ)⊕Xc⊕a⋅Δ\sH(X_b\oplus\Delta)\oplus X_c\oplus a\cdot\DeltaH(Xb​⊕Δ)⊕Xc​⊕a⋅Δ

These two ciphertexts are then suitably permuted according to the color bits of XbX_bXb​. The evaluator takes a hash of its wire label for XbX_bXb​ and decrypts the appropriate ciphertext. If a=0a = 0a=0, he/she obtains output wire label XcX_cXc​ in both values of bbb. If a=1a = 1a=1 the evaluator obtains either XcX_cXc​ or Xc⊕ΔX_c\oplus \DeltaXc​⊕Δ, depending on the bit bbb. Intuitively, the evaluator will never know both XbX_bXb​ and Xb⊕ΔX_b\oplus\DeltaXb​⊕Δ, hence the other ciphertext appears completely random.

By applying the row-reduction optimization, we reduce the number of ciphertexts from 222 to 111 as follows. H(Xb⊕Δ)⊕H(Xb)⊕a⋅Δ\sH(X_b\oplus\Delta)\oplus \sH(X_b)\oplus a\cdot\DeltaH(Xb​⊕Δ)⊕H(Xb​)⊕a⋅Δ

Evaluator half-gate. Considering the case of an AND\andAND gate a∧b=ca\wedge b = ca∧b=c, where a,ba,ba,b are intermediate wires in the circuit and the evaluator somehow knows the value aaa at the time of evaluation. The evaluator can behave differently based on the value of aaa.

When a=0a = 0a=0, the evaluator should always obtain output wire label XcX_cXc​, then the garbled circuit should contains the ciphertext: H(Xa)⊕Xc\sH(X_a)\oplus X_cH(Xa​)⊕Xc​

When a=1a = 1a=1, it is enough for the evaluator to obtain R=Xc⊕XbR= X_c\oplus X_bR=Xc​⊕Xb​. He/she can then XOR\xorXOR RRR with the other wire label (either XbX_bXb​ or Xb⊕ΔX_b\oplus \DeltaXb​⊕Δ) to obtain either XcX_cXc​ or Xc⊕ΔX_c\oplus \DeltaXc​⊕Δ. Hence the garbler should provide the ciphertext: H(Xa⊕Δ)⊕Xc⊕Xb\sH(X_a\oplus\Delta)\oplus X_c\oplus X_bH(Xa​⊕Δ)⊕Xc​⊕Xb​

Combining the above two case together, the garbler should provide two ciphertexts: H(Xa)⊕Xc\sH(X_a)\oplus X_cH(Xa​)⊕Xc​ H(Xa⊕Δ)⊕Xc⊕Xb\sH(X_a\oplus\Delta)\oplus X_c\oplus X_bH(Xa​⊕Δ)⊕Xc​⊕Xb​

Note that these two ciphertext do NOT have to be permuted according to the color bit of XaX_aXa​, because the evaluator already knows aaa. If a=0a = 0a=0, the evaluator uses the wire label XaX_aXa​ to decrypt the first ciphertext. If a=1a = 1a=1, the evaluator uses the wire label Xa⊕ΔX_a\oplus \DeltaXa​⊕Δ to decrypt the second ciphertext and XOR\xorXORs the result with the wire label for bbb.

By applying the row-reduction optimization, we reduce the number of ciphertexts from 222 to 111 as follows. H(Xa⊕Δ)⊕H(Xa)⊕Xb\sH(X_a\oplus\Delta)\oplus \sH(X_a)\oplus X_bH(Xa​⊕Δ)⊕H(Xa​)⊕Xb​

Two halves make a whole. Considering the case to garble an AND\andAND gate a∧b=ca\wedge b = ca∧b=c, where both inputs are secret. Consider: c=a∧b=a∧(r⊕r⊕b)=(a∧r)⊕(a∧(r⊕b))c = a \wedge b = a\wedge (r\oplus r\oplus b) = (a\wedge r)\oplus (a\wedge(r\oplus b))c=a∧b=a∧(r⊕r⊕b)=(a∧r)⊕(a∧(r⊕b))

Suppose the garbler chooses uniformly random bit rrr. In this case, the first AND\andAND gate (a∧r)(a\wedge r)(a∧r) can be garbled with a garbler-half-gate. If we further arrange for the evaluator to learn the value r⊕br\oplus br⊕b, then the second AND\andAND gate (a∧(r⊕b))(a\wedge(r\oplus b))(a∧(r⊕b)) can be garbled with an evaluator-half-gate. Leaking this extra bit r⊕br\oplus br⊕b to the evaluator is safe, as it carries no information about the sensitive value bbb. The remaining XOR\xorXOR gate is free and the total cost is 222 ciphertexts.

Actually the evaluator could learn r⊕br\oplus br⊕b without any overhead. The garbler choose the color bit of the 0 label XbX_bXb​ on wire bbb. Since that color bit is chosen uniformly, it is secure to use it. Then when a particular value bbb is on that wire, the evaluator will hold a wire label whose color bit is b⊕rb\oplus rb⊕r.

Let Xb0,Xb1=Xb0⊕ΔX_b^0,X_b^1 = X_b^0\oplus \DeltaXb0​,Xb1​=Xb0​⊕Δ be the labels for bbb, let p=LSB(Xb0)p = \lsb(X_b^0)p=LSB(Xb0​). ppp is the color bit of the wire, is a secret known only to the garbler. When the evaluator holds a wire label for bbb whose color bit is sss, the label is Xbs⊕pX_b^{s\oplus p}Xbs⊕p​, corresponding to truth value s⊕ps\oplus ps⊕p.
